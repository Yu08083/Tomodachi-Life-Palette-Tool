# すぽいと帳

🌍 日本語 / English / 한국어 / Français / Español / 繁體中文 / 简体中文 / Deutsch / Italiano / Nederlands / Português (BR) / Русский

[![技術者倫理 遵守済み](https://img.shields.io/badge/%E6%8A%80%E8%A1%93%E8%80%85%E5%80%AB%E7%90%86-%E9%81%B5%E5%AE%88%E6%B8%88%E3%81%BF-0a0a0a?style=for-the-badge&labelColor=ffffff)](https://技術者倫理.com)

トモダチコレクション わくわく生活のペイント機能で使える、画像 → ドット絵 変換ツール。

任意の画像をゲーム内 84 色パレットに近似変換し、HSV 操作手順や色番号塗り絵まで生成します。

オンライン版: <https://yu08083.github.io/Tomodachi-Life-Palette-Tool/>

## 主な機能

### 入力

- ドラッグ&ドロップ または クリックでアップロード（PNG / JPG / GIF / WEBP）
- 以前作った作品の **`.spoito.json`** ファイルからも復元可能
- 用途に合わせた **5 種のアスペクト比** からクロップを選択:
  - **1:1**（256×256）— 通常のプロフィール画像
  - **本** 180×256 — 縦長ブック
  - **TV** 256×131 — テレビ画面
  - **ゲーム** 256×144 — ゲーム画面風
  - **壁紙** 172×256 — 縦長壁紙
- ハンドルでサイズ変更、内側ドラッグで位置調整。**スマホはハンドルが大きめに自動調整**
- 上下/左右反転・90° 回転にも対応
- いつでも「クロップやり直し」で再クロップ
- **履歴機能** — 最近開いた画像はサムネで一覧表示、ワンタップで復元

### 変換

- ゲーム内 84 色パレットへの自動変換（ディザ ON/OFF 切替）
- 2 系統のブラシサイズ:
  - **ゲーム準拠**（Pixel-perfect）— 4px / 8px / 16px / 32px ・ ゲーム内ブラシと一致
  - **中間サイズ**（Smooth）— 1px / 2px / 3px / 7px / 13px / 19px / 27px ・ より細かい刻み
- **明度・彩度スライダー** で変換前に画像を微調整、ワンクリックでリセット

### 表示

- ピクセル単位のクリック / タップで色取得（HEX / RGB / HSV）
- **大きな行/列ピル** — クリック位置を「13行9列 / #393/1024」のように常時表示、塗ってる最中の視線移動を削減
- ゲーム内 HSV 操作手順の自動算出
- 使用色レシピ — 各色のセル数、占有率、行列位置を一覧
- **塗り終わり管理** — 完了した色をワンタップでマーク、レシピで取り消し線表示、キャンバスでフェードアウト
- **塗りやすさスコア** — 1〜100 の点数 + 5 段階カラーバッジ（入門・初級・中級・上級・達人）で難易度の目安
- **アイソレート（色だけ表示）** — 選択中の色だけハイライトして残りをフェード、塗る場所が一目で分かる
- **対称（ミラー）モード** — 縦軸 / 横軸 / 4 方向 から選択、選択位置の鏡像をハイライト。顔やシンメトリーモチーフに便利
- パレット ↔ レシピの hover ハイライト連動
- HEX コード直接入力で色をパレット上に位置表示（**1–84 のパレット番号でもジャンプ可能**）
- 「使用色のみ」フィルタでパレットを絞り込み（完了済みも自動で除外、残作業が一目で分かる）
- **グリッド表示** — 1 セル / 8 セル区切り / 中央線 の階層的オーバーレイ、8/16 ステップ切替
- **外部ルーラー** — 行 / 列の位置番号を欄外表示、ズーム・拡張比率に応じて目盛間隔を自動調整
- お気に入り色を 24 件まで保存・呼び出し
- 1× 以下のズームアウト（0.1× まで）と 16× までのズームイン

### 出力

- **ドット絵 PNG** — ロゴ・タイトル帯付き、SNS シェア向け
- **番号塗り絵 PNG**（Paint-by-Number）— 各セルに色番号 + 凡例付き、印刷して手塗り可能
- **タイムラプス録画**（WebM）— 塗り作業の進捗を動画として保存、SNS シェア用に。MediaRecorder で実装、追加ライブラリ不要
- **作品共有 URL / QR コード** — 現在の状態を URL に埋め込んで別端末で開ける（QR を生成してスマホで読み取り可）
- **`.spoito.json` 保存** — 編集状態を残してあとで続きから

# 更新履歴

ここ最近のアップデートまとめです。バグ修正の方が多いんだけど、新しい機能もいくつか入れた。

## あの後やった怒涛のターン

最後の更新からだいぶ詰め込んだので一気にまとめる。

### プリセット背景に画像を貼る機能

ゲーム内で「服や帽子」の色塗りをしたい時に、目当ての画像をその領域だけにフィットさせて 84 色変換できるやつ。プリセットは 4 カテゴリ × 4 種(トップス1・トップス2・ボトムス・帽子) = 16 種類。

仕組み:

1. プリセット PNG の中の「ペイント対象」の青いエリア (RGB 5,75,186) を自動検出
2. 領域ごとに「ここに画像置いてね」UI を表示
3. ユーザーがアップロード → ドラッグ移動 + ホイール / ピンチで拡縮
4. 「配置確定」で領域マスクに沿って合成 → 84 色変換 → 出力

色マッチングは **Lab 色空間** で ΔE76 距離で計算。RGB だと「ピンクと紫」みたいな色が混乱するので、人間の知覚に近い Lab に変換してから比較してる。Floyd-Steinberg ディザのオン/オフ切替も。

スマホでは領域選択 → ファイル選択の流れがスムーズになるよう挙動を全部やり直した。最初はクロップツール経由してたんだけど、毎回切り出すのは面倒なので「**領域内に画像を置く」モード**を作って一段階減らした。

配置中の画像にも明度・コントラスト・彩度の補正パネル付き(ゲーム内で薄め/濃いめに見せたい時用)。

### ピクセルエディタ

変換後のドット絵を**直接ピクセル単位で編集**できるようにした。84 色パレット縛りなので、ゲームで描ける絵から外れない。

ツール:

- ✏️ 鉛筆 — ドラッグでなぞった全マス塗る
- 🧽 消しゴム — 同上、透明化
- 💧 スポイト — タップしたマスの色を「使う色」にセット
- 🪣 バケツ — 4 連結フラッドフィル
- 🔁 色置換 — タップしたマスと同じ色を全部新色に
- ⤺⤻⇄⇅ 回転・反転 — 90° 左右 / 水平 / 垂直
- ↶ Undo / ↷ Redo(Ctrl+Z / Ctrl+Y、50 ステップ)
- ⟲ Reset — 編集開始時のスナップショットに戻る

ドラッグ中は Bresenham で線を引いてマス飛ばし防止、`setPointerCapture` で離した場所がキャンバス外でも追従する。

色の選び方は最初「パレットからクリック」に強制してたけど、yu0 が「**使う色のスウォッチ自体クリックできなくね?**」と言ってきて確かにってなった。スウォッチクリックで 12×7 のミニパレットがポップアップ、その場で選べる。Esc / 外側クリックで閉じる。

変換データ自体を破壊するので、変換タブ切替時は警告して、外に出る時は state を保持。

### カメラ撮影

スマホ実機で動かす時、いちいち写真撮ってアップロードするのが面倒なので、ブラウザから直撮りできるように。`getUserMedia` で前面 / 背面カメラ切替、自撮りは左右反転プレビュー、撮影 → クロップツールに直行。

`getUserMedia` 非対応のブラウザでは「カメラで撮影」ボタンが自動で消える。

### 画像補正(明度・コントラスト・彩度・シャープ)

クロップ画面でスライダー 4 本で原画を整える。やりたかったこと:「写真を撮ったけど暗いから明るくしてからドット絵化したい」みたいなケース。

最初は cropSrcCanvas を直接書き換えてリアルタイム反映する設計にしたんだけど、効かなかった。原因よく分からん。3 回くらい書き直して、最終的に「**`drawCropCanvas` の描画時に `ctx.filter` を適用、確定時に bake する**」方式に落ち着いた。これでブラウザの native canvas filter API が直接効くので、原因不明の謎挙動はなくなった。

シャープは custom 3×3 アンシャープマスク自前実装(`ctx.filter: sharpen()` は無い)。bake 時にだけ適用、リアルタイム反映は省略(重いので)。

### シェアカード(1080×1080 PNG)

完成した作品を SNS にシェアする時用に、専用カードを生成する機能。1080×1080 PNG で

- 上にロゴ + サイトタイトル
- 真ん中にドット絵作品(白いカード型背景)
- 下に統計(サイズ / ピクセル数 / 使用色数 / 難易度)
- 日付 + サイト URL

このカード生成、最初は共有モーダルの中に「シェアカード」セクションとして埋め込んでたんだけど、yu0 が「**履歴アイテムの横に置くべきじゃない?**」と提案。確かに「過去作をシェア」したい時に、いちいち復元して現在の作業を上書きするのは UX が悪い。

履歴の各アイテム横に 📤 ボタン追加 → クリックで現在の作業状態に**全く触れず**にその履歴のシェアカードを生成・表示。`convertImage` をその場で再実行して一時的な convertedData 構築 → 専用モーダルで表示。

### 「最初に戻る」ボタン

メインビューのヘッダー(選択/移動の隣)に追加。クリックで全 state クリアして「画像をアップロード」画面に戻る。既存の「別の画像を読み込む」と同じ動作だけど、ヘッダー位置で常時アクセス可能。

### PWA(Service Worker)

`manifest.json` は前からあったんだけど、「真の PWA(オフライン動作・Chrome のインストールボタン)」には Service Worker が必要だった。50 行くらいの `sw.js` 追加して:

- インストール時にアプリシェルキャッシュ
- フェッチは stale-while-revalidate(キャッシュ即返し + 裏で更新)
- オフラインでも全機能動く
- Chrome デスクトップの URL バー右に「**インストール**」ボタン自動表示

スマホもホーム画面追加した時の挙動が本格的に。

### console art 全面リニューアル

DevTools の Console を開いたら出てくる ASCII アート + パレット表示 + コマンド集。

最初は AI 感満載で「`👋 Good evening, curious developer!`」みたいな挨拶が出てたんだけど、yu0 に「**AI 感を消す**」と指摘されて削除。日本語の時間別挨拶のみに(「夜更かしお疲れさま」「こんばんは」等)。

パレット表示も顔(Mii のドット絵)を削って、代わりに「**猫と和解せよ**」のバナーを 6 色装飾で追加。Mii の方は権利的に怪しい絵柄だったので、これで気兼ねなく公開できる。

パレット格子のセル位置も「行ごとに別の console.log 呼び出し」方式に変えたら綺麗に揃った。1 回の log に \n で複数行入れるとブラウザがインデント入れることがあって、これがズレ原因だったらしい。

文字色も初期は濃ブラウンで Chrome の dark mode console で読めなかったので、暖色系ミディアム(`#D9B891`)に統一。両モード OK。

### 大画像の自動縮小

スマホで撮った 4032×3024 みたいなデカ写真を投げると、原画ビューでキャンバスが画面からはみ出してた。1600px 超は内部で縮小して扱うように。

「画質劣化」を yu0 に心配されたけど、ドット絵への変換結果(64×64)には影響ゼロ。1600px から 64×64 にサンプリングする時の元ピクセル数は 25×18 ≒ 450、十分すぎる。劣化するのは「原画を画面で眺める時の見え方」だけで、`imageSmoothingQuality='high'` 適用してるのでこれもほぼ気付かないレベル。

### Lighthouse スコア上げ

定期チェックタイム。

- 全 23 個の JS に `defer` 付加 — レンダリングブロッキング解消、FCP / LCP 大幅短縮
- PNG 全圧縮(ImageMagick PNG8 indexed mode、256 色)— 合計 640KB 削減、視覚劣化ゼロ
- `<meta name="color-scheme" content="light dark">` 追加 — ブラウザに light/dark モード対応を明示
- canvas に `willReadFrequently: true` — DevTools の警告 + 微妙な高速化

23 個の `<script>` が defer 化すると、最初のレンダリング表示までが体感速くなる。

## まず壊れてたところ

### 「使用色のみ」と「塗り終わった」がケンカしてた

レシピで色をチェックしても、パレット側は何も変わらないままで、「使用色のみ」フィルタを ON にしてもチェック済みの色がふつうに明るく表示されてた。完了してるかどうかが視覚的に分からなくて、せっかくの完了マークが活きてなかった。

直すついでに、「使用色のみ ON のとき、完了済みは薄白で表示」というルールにした。塗っていくと残ってる色だけがパレットに残るので、進捗が目に見えて気持ちいい。

### 原画ビューで完了マークしても reset が出てこない

別件で気づいたんだけど、原画ビューで「塗り終わった」を押しても、リセットボタンが現れない問題があった。

中を見たら、完了状態の管理コードが 3 ファイルにまたがってて、状態を書き換えても UI 更新があちこちで抜けてた。`recipe.js` `canvas-render.js` `color-info.js` で似たような関数が重複してたり、`recipeCheckState` を直接いじってたり。

これは小手先で直すより整理した方が早いと思って、`doneState` っていう小さい API を作ってまとめた:

```js
doneState.isDone(idx)
doneState.toggle(idx)
doneState.reset()
doneState.count()
doneState._notify()  // 状態変更で関連UIを一括更新
```

これで状態を変えるルートが 1 本になって、UI 更新の漏れも消えた。ついでに画像ロード時に変換データを先に作るようにして、原画ビューでもレシピと reset ボタンが最初から使えるようになった。

### チェックボックスに変な線が出る

スマホで「塗り終わった」ボタンの左をタップした時に、レシピのチェックボックスに「/」みたいな線が一本うっすら出る現象。最初なんの線か分からなくて 30 分くらい原因探した。

正体は、`✓` を CSS で表現する時の小細工だった。12×7px の長方形を -45deg 回転させて、下と左のボーダーだけ白くすることで `✓` の形を作ってたんだけど、透明ボーダーの subpixel rendering で稀に一本だけ線が残ってしまってた。

回転矩形やめて、Unicode の `✓` を `::after` で出すように変更。シンプル is the best。

### 選択ピルが画像と被ってた

クリックした位置を「13行9列」みたいに大きく表示する floating pill を作ったんだけど、最初キャンバスの右上に absolute で置いてた。これが当然のように画像と被って邪魔だった。

キャンバスの **上** に独立した行として配置するように変更。常に見えて、画像と被らない。

## 新しく入れた機能

### 行/列ピル

画像のどこをクリックしたか、いちいちカラーパネルに目を移さずに「**13行9列 / #393/1024**」と大きく表示。塗ってる最中の視線移動が減る、地味に効くやつ。

### 対称(ミラー)モード

顔とか左右対称のモチーフを描く時に、片側で選んだ色の **鏡像位置をハイライト** してくれる。縦軸/横軸/4方向の 3 モード。

ヘッダーに「対称」ボタンを追加。サブトグルで `↔ ↕ ✚` を選べる。設定は localStorage に保存。

### 塗りやすさスコア(100段階)

最初 ★1〜5 で実装してたんだけど、星 5 段階だと粒度が荒すぎて「同じ★3でも全然違うじゃん」となるので 1〜100 にした。スコア + 色分けバッジ + ?ボタンで説明モーダルって構成。

色は緑(入門)→ 青(初級)→ 黄(中級)→ 橙(上級)→ 赤(達人)。スコアは「使用色数」と「色の散らばり具合」から計算してる。あくまで目安。

### タイムラプス録画

塗り作業の進捗を WebM 動画として保存できる。MediaRecorder + canvas.captureStream で実装、外部ライブラリ無依存。

録画中はボタンが赤くなって ● が点滅 + 経過時間表示。停止すると自動でダウンロード。

GitHub Pages (HTTPS) では普通に動くけど、`file://` で直接開くと MediaRecorder のセキュアコンテキスト要件でブロックされる。ローカルテストなら `python3 -m http.server` などで OK。

### スマホクロップのハンドルが指で掴みやすく

タッチデバイス検出して、ハンドルの見た目を 14 → 18px、ヒット範囲を 18 → 26px に拡大。指で掴むのが楽になった、はず。

## SEO もりもり

ここからは検索流入のためのテクニカルなやつ。

### メタタグ拡張

`description` と `keywords` に Mii、似顔絵、帽子、服、床絵、看板、3DS、Nintendo、quantize、generator... と関連語を盛り込んだ。やりすぎると逆効果なので健全な範囲で。

`og:` / `twitter:` も同じく更新、画像 URL に `?v=3` 付けてキャッシュバスト。

### 構造化データ(JSON-LD)を 4 種類

- `WebApplication` — 既存
- `WebSite` — 自分の X / GitHub / Discord を `sameAs` で結びつけ
- `FAQPage` — Q&A 7つ。検索結果にアコーディオン表示される可能性
- `HowTo` — 5ステップの使い方。番号付きで出る可能性

「可能性」と書いてるのは、Google が表示するかは Google が決めるから。出れば儲けもん。

### 言語切り替え時に title / description も切り替え

12言語対応してるのに `<title>` と `meta description` がずっと日本語固定だった。英語ページとして検索された時に日本語のタイトルが出るのは弱い。

言語切替で `<html lang>` `<title>` `<meta description>` `og:title` `og:description` `twitter:title` `twitter:description` ぜんぶ切り替わるようにした(コード自体は既存だったけど i18n 辞書の値が短いままだったので、12言語ぶん書き直し)。

### PWA 化

`manifest.json` 追加して、Apple / Mobile Web App メタも完備。スマホでホーム追加するとアプリっぽく起動できる。検索結果でも多少優遇されるはず。

### 可視 SEO セクション

Google は metaタグより本文を重視するので、トップページの下部に「このツールは何ができるか」を 12言語で書いたセクションを追加。デザイン的にも宣伝になるから一石二鳥。
